# DM-balanceV1 自瞄任务数据流与操作指南 (H7_new)

聚焦自瞄目标处理与云台/射击联动。

## 主要文件与职责
- 模块：`User/APP/auto_aim.[ch]`
  - 接收通信传入的目标信息，计算云台目标角与开火许可。
- 依赖：`Communication/core`（数据输入）、`gimbal_task`（消费目标）、`shoot`（根据锁定/许可开火）、`INS_task`（姿态）、`remote_control`（模式开关）。

## 数据流
1) 输入
   - 目标包：来自通信栈（上位机/视觉板），包含 yaw/pitch 偏差、距离、置信度等。
   - 模式开关：遥控自瞄使能。
   - 姿态/位姿：来自 `INS_task`，用于坐标转换或补偿。

2) 处理流程
   - 校验数据：置信度/时间戳/丢包判断。
   - 计算目标角：将目标偏差转换为绝对 yaw/pitch 设定，考虑当前姿态。
   - 判定是否允许开火：根据锁定状态、置信度、热量/功率（可读 referee 数据）。
   - 输出：目标 yaw/pitch、开火许可标志。

3) 输出对接
   - 云台：`gimbal_behaviour` 读取目标，切入自瞄模式并跟踪。
   - 射击：`shoot` 读取许可/锁定，控制进给时机。
   - 通信反馈：可回传锁定状态/延迟/命中（经 Communication 栈）。

## 典型链路
- 视觉板 → Communication/core → auto_aim 解析 → gimbal 自瞄模式 → PID 跟踪 → shoot 读取许可开火。

## 关键配置
- 置信度/超时阈值：在 `auto_aim` 内部常量或宏。
- 坐标系/符号约定：需与上位机一致，必要时在转换时加符号调整。
- 机器人类型/云台参数：使用 `robot_param` 中的限幅与方向。

## 调试建议
- 先固定云台，注入假目标，检查 yaw/pitch 指令方向是否正确。
- 记录自瞄输入与云台输出，排查延迟与抖动；必要时低通滤波目标。
- 与射击联调时，先关闭发弹，只看许可逻辑是否随锁定变化。
