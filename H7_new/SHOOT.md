# DM-balanceV1 射击任务数据流与操作指南 (H7_new)

聚焦射击（3508/双发）模块的任务位置、状态机、输入输出与调试要点。

## 主要文件与职责
- 任务/状态：`User/APP/shoot.[ch]`（共用接口与状态机框架）
- 具体机构：
  - 单/3508：`User/shoot/shoot_3508.[ch]`
  - 双拨盘/双层：`User/shoot/shoot_double.[ch]`
- 参数：`User/APP/robot_param*.h`（摩擦轮/拨盘 CAN ID、方向、目标转速、PID）
- 依赖：`remote_control`（触发）、`auto_aim`（开火许可/锁定）、`referee`（热量/功率限制）、`can_bsp`（下发）、`Algorithm/PID`

## 数据流
1) 输入
   - 遥控射击开关/射频：来自 `remote_control`
   - 自瞄许可：来自 `auto_aim`（锁定/允许开火）
   - 热量/能量：`referee` 提供剩余热量/功率限制
   - 电机反馈：摩擦轮/拨盘编码器、转速、电流

2) 状态机（`shoot` 框架）
   - IDLE/STOP → 摩擦轮预热 → 允许发射 → 进给/清弹 → 超热/故障回退
   - 双发/单发：根据机构选择对应实现

3) 目标与控制
   - 摩擦轮：目标 RPM（遥控档位或固定值）；速度环 PID 控制电流
   - 拨盘/给弹：位置/速度控制；卡弹检测（电流/速度异常）触发反转/抖动
   - 热量限制：若超限，暂停进给或降速

4) 下发/输出
   - CAN 电流：摩擦轮、拨盘（双发则双通道）
   - 状态上报：当前模式、热量、转速、卡弹标志供 UI/通信

## 典型链路
- 手动射击：DBUS 开关 → `shoot` 状态机 → 摩擦轮转速 PID → 拨盘步进 → CAN。
- 自瞄射击：自瞄锁定+许可 → `shoot` 状态机 → 自动进给 → CAN；并可通知 gimbal/comm 状态。
- 双发：在 `shoot_double` 中协调两路拨盘，确保同步与防卡弹。

## 关键配置
- CAN ID/方向/齿比：`robot_param*.h`
- 目标转速、限速、卡弹阈值：同上或在各模块内部常量
- 热量/功率策略：在 `shoot` 逻辑中引用裁判数据，可按需要调整策略

## 调试建议
- 先独立测试摩擦轮，确认转速和电流方向正确。
- 拨盘空载跑位置/速度环，验证编码器方向，再带摩擦轮低速试射。
- 卡弹检测阈值需结合电机电流与机械阻尼调节，避免误判。
- 自瞄模式下，确保开火许可逻辑正确防止误发。
